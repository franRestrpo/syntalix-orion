feat: Implementación de mejoras de seguridad y calidad de código v2.0

## Resumen
Implementación de las mejoras de seguridad, validación y calidad de código
recomendadas en el análisis del proyecto. Esta actualización incluye módulos
centralizados de seguridad, validación con Pydantic, logging estructurado y
suite completa de tests unitarios.

## Cambios Principales

### SEGURIDAD

#### Nuevo módulo de seguridad centralizado
- core/security.py: Módulo completo con:
  - SecurityConfig: Singleton para gestión de contexto SSL configurable
  - generate_secure_password(): Contraseñas criptográficamente seguras
  - hash_password_bcrypt()/verify_password_bcrypt(): Hashing de contraseñas
  - generate_api_key(): Generación de API keys seguras
  - validate_domain()/validate_email(): Validación de formatos
  - sanitize_input(): Protección contra inyección de comandos
  - mask_secret(): Enmascaramiento seguro para logging

#### Eliminación de vulnerabilidad SSL
- validate_swarm.py: 
  - Eliminación de verify=False (vulnerable a MITM)
  - Implementación de contexto SSL configurable con CA bundle
  - Parámetros verify_ssl y ca_bundle configurables
  - Verificación real de certificados SSL
  - Integración con módulo de seguridad

#### Eliminación de credencial débil
- procesar_aplicacion.yml:
  - ELIMINADO: 'admin123' hardcodeado como password por defecto
  - NUEVO: Generador de passwords de 32 caracteres seguros
  - Usa lookup('password', '/dev/null', length=32, chars='ascii_letters,digits')

### VALIDACIÓN Y MODELOS

#### Modelos Pydantic
- core/models.py: Nuevos modelos con validación:
  - AppVariable: Variables de entorno tipadas
  - AppMetadata: Metadatos de aplicaciones validados
  - DeploymentPlan: Planes de despliegue con validación
  - load_app_catalog(): Carga validada de catálogo completo
  - Validaciones: tipos, categorías, longitudes, dependencias

### LOGGING

#### Logging estructurado
- core/logging_config.py: Sistema completo de logging:
  - OrionLogger: Logger singleton configurado
  - get_logger(): Helper para obtener loggers
  - JSONFormatter: Formato JSON para parsing automático
  - StructuredFormatter: Formato legible con colores
  - LogContext: Context manager para datos extra
  - Rotación automática (10MB, 5 backups)

### TESTS

#### Suite de tests unitarios
- tests/test_security.py: 30 tests para módulo de seguridad
- tests/test_models.py: 20 tests para validación Pydantic  
- tests/test_dependency_graph.py: 18 tests para grafo de dependencias
- tests/conftest.py: Fixtures y configuración pytest

### DEPENDENCY GRAPH

#### Mejoras integradas
- core/dependency_graph.py:
  - Integración con logging estructurado
  - Carga inteligente de APP_METADATA con fallback
  - Generación de secretos integrada
  - Nuevo método validate_plan() para validación

### UTILS

#### Utilidades de seguridad
- utils.py:
  - check_docker_available(): Verificar Docker
  - check_swarm_active(): Verificar Swarm
  - check_network_exists(): Verificar redes Docker
  - create_overlay_network(): Crear redes overlay
  - generate_app_password(): Generador de passwords seguros
  - validate_app_domain()/validate_app_email(): Validación
  - load_env_file()/save_env_file(): Gestión segura de .env

## Uso

# Instalar dependencias
cd Orion-Python-Ansible/scripts
pip install -r requirements.txt

# Ejecutar tests
pytest -v --cov=core --cov-report=html

# Generar password seguro
python -c "from core.security import generate_secure_password; print(generate_secure_password())"

# Configurar logging
python -c "from core.logging_config import setup_logging; setup_logging('INFO')"
