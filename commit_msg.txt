feat: Implementación del Playbook Maestro site.yml - Syntalix-Orion V2

## Resumen
Creación del playbook maestro de orquestación para Syntalix-Orion V2.
Este playbook ejecuta dinámicamente roles según las aplicaciones
seleccionadas desde la TUI.

## Cambios Principales

### 1. Playbook Maestro: site.yml

**Archivo: site.yml** (~350 líneas)

Playbook principal que:
- Valida que `ansible_enabled_roles` esté definido (seguridad)
- Verifica/crea la red Docker overlay `SyntalixNet`
- Ejecuta roles condicionalmente según selección del usuario
- Muestra resumen final del despliegue

**Estructura**:
```yaml
- name: "Syntalix-Orion V2 - Despliegue de Infraestructura"
  hosts: all
  become: yes
  gather_facts: yes

  pre_tasks:
    - Validación de seguridad (assert)
    - Verificación de red Docker
    - Resumen pre-despliegue

  roles:
    # 28 roles mapeados con condiciones when:
    # Core (4), Data (7), Monitoring (4), AI (3)
    # Automation (2), Communication (3), Management (4)

  post_tasks:
    - Resumen final
    - URLs de acceso
    - Comandos útiles
```

## Flujo de Ejecución

```
┌──────────────────────────────────────────────────────────────┐
│                       TUI (tui.py)                           │
│  1. Usuario selecciona apps                                 │
│  2. DependencyGraph.plan_with_vars_multi()                  │
│  3. Genera ansible_vars.yml con secrets seguros             │
│  4. Contiene: ansible_enabled_roles = ["traefik", "n8n"]    │
└─────────────────────────┬────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────┐
│                   ansible-playbook                          │
│  ansible-playbook -i inventory site.yml -e @ansible_vars.yml│
└─────────────────────────┬────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────┐
│                     site.yml                                │
│                                                              │
│  pre_tasks:                                                  │
│    - assert: ansible_enabled_roles defined?                  │
│    - Verificar/Crear red SyntalixNet                        │
│                                                              │
│  roles:                                                      │
│    - core/traefik (when: "'traefik' in ansible_enabled_roles")│
│    - apps_automation/n8n (when: "'n8n' in ansible_enabled_roles")
│                                                              │
│  post_tasks:                                                 │
│    - Mostrar resumen final                                  │
└──────────────────────────────────────────────────────────────┘
```

## Validación de Seguridad

El playbook valida que:
1. `ansible_enabled_roles` esté definido
2. La lista no esté vacía

Si no se cumple, muestra error instructivo:
```
ERROR: La variable 'ansible_enabled_roles' no está definida.
SOLUCIÓN: Ejecuta con -e @ansible_vars.yml
```
